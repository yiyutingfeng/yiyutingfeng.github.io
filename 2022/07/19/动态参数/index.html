    动态参数

    发表于 2022-07-19 更新于 2023-05-03 本文字数： 767 阅读时长 ≈ 3 分钟

    无需重启CirroEngine,通过SQL修改默认参数值

    目前支持修改的参数

    DB级参数 说明 参数类型 默认值 参数值范围
    page_compress_type Page 数据压缩算法：1-SNAPPY，2-LZ4；如果使用非SNAPPY 算法，会影响与标准parquet 格式的兼容 int32 1 {1,2}
    datapack_maxsize 单个 DataPack 文件大小上限，单位Byte int64 1073741824 [223,263-1]
    datapack_maxrownum 单个 DataPack 文件数据行数上限 int64 10485760 [1,263-1]
    dictpage_num ColumnChunk 中字典页的缓存页数，内存中字典编码的 buffer 大小为 dictpage_num * page_size，值越大需要的内存越多，编码效率更高 int32 4 [1,4]
    filesize_reserverate DP 文件中实际使用空间占 datapack_maxsize 规定的单个 dp 文件大小的百分比，dp 文件 footer 的估值并不十分准确，为了确保 dp 文件的size 不会超过 datapack_maxsize，给 datapack_maxsize 乘个百分比作为 dp 文件中存储的数据大小 int32 90 (0,100]
    no_page_compress_optimize 是否禁止 Page 压缩优化；如果不禁止，会影响与标准 parquet 格式的兼容 bool TRUE TRUE/FALSE
    compress_validrate DP 文件数据压缩百分比阈值，当 no_page_compress_optimize 为 false 时才会生效，压缩后与压缩前大小比例超过该值才进行压缩 int32 60 [1,100]
    dictcardinality 字典编码中，字典条目上限，字典条目达到参数指定条数或者字典编码的 buffer 占满，则写字典页 int32 65535 (0,231-1]
    plain_encoding_rule 指定采用默认 Plain 编码规则的数据类型，多个类型用’,’分隔，如果数据类型指定了 plain 编码规则，该类型不生成字典页 string 空 {int, long, double, date, yminterval, dsinterval, timestamp, number, char, char(n), varchar, varch(n)}中选择一个或多个
    page_rownum_index 是否在加载数据时，创建 page_row 索引，点查询优化需要该参数值为 true bool TRUE TRUE/FALSE
    page_max_min_index 是否在加载数据时，创建 page 内部极值索引，点查询优化需要该参数值为 true bool TRUE TRUE/FALSE
    PC级参数 说明 参数类型 默认值 参数值范围
    write_buffer_num 异步加载线程使用的 MDP 队列中 mdp 的数量，参数原名为 mdp_queue_num int32 3 [1,5]
    cast_timestamp_format Timestamp类型默认格式，小数精度为9 string YYYY-MM-DD HH24:MI:SS.FF
    cast_date_format Date类型默认格式 string YYYY/MM/DD HH24:MI:SS
    compact_group_num 整并分组数 int32 1 [1,64]
    compact_filesize_threshold 不参加整并的文件大小下限，单位BYTE int64 849346560 [0,263-1]
    local_data_cache_policy 本地数据缓存策略.NO_CACHE: 不进行本地缓存 AUTO_CACHE: 自动选择节点进行缓存（最多选择两个） GLOBAL_CACHE: 所有节点都可以缓存 string AUTO_CACHE NO_CACHE, AUTO_CACHE, GLOBAL_CACHE
